# Design Direction — Personal Finance CFO Agent Local App

Status: **locked for MVP v0.1**. This records the design decision from the Claude
Design pass so Codex / Claude Code have a single implementation spec.

The visual exploration artifact (3 directions × 7 screens, one pannable canvas)
was produced in Claude Design. A PDF export (`CFO Agent - UI Directions.pdf`) is
kept locally and is intentionally not committed; re-export a complete PDF/PPTX
from Claude Design if a tracked copy is needed (the current PDF export clips the
third column).

## 1. Chosen direction

**Calm CFO Dashboard** is the primary shell.

- Feel: premium, quiet, executive, trusted-advisor.
- Passes the 10-second test on the Home Dashboard.
- Best for portfolio screenshots and the normal monthly check-in.

Do **not** build three separate apps. Instead:

- **Analyst Workbench** becomes a dense-table *mode* used inside Category Review,
  Stress Test Explorer, and source/audit views.
- **Personal Finance Coach** becomes a *tone layer* and the home for the future
  optional local AI memo.

## 2. Visual system

- **Mode:** light/premium for the Calm shell. Dark, monospace tables only where
  audit density helps (Workbench mode).
- **Type:** refined serif headings + clean sans body, with **tabular numbers**.
  Monospace only for audit/source labels, artifact IDs, and technical status.
- **Charts:** moderate. Numbers lead, charts explain trends (cash projection,
  budget vs actual), tables support audit. No decorative chart clutter.
- **Persona for all sample screens:** one dual-income household saving toward a
  first home (the complex-household fixture). Shows rent-vs-buy, home-purchase readiness,
  goals, risks, cash runway, and monthly planning.

### Trust components (non-negotiable)

- **Privacy/status banner:** sample mode active, real data locked, local-only, no
  bank login, no cloud sync, no cloud AI, local AI off by default.
- **Verified marker:** "Verified by engine" on every calculated figure.
- **Local-AI source label:** when (and only when) AI text appears, quarantine it
  in a clearly separated block — "Generated by local AI", "Numbers calculated by
  deterministic engine", "Based on verified artifacts: ...". No cloud fallback.

## 3. MVP v0.1 — "Read, Trust & Upload"

Streamlit over the report JSON contract (`docs/REPORT_JSON_CONTRACT.md`) plus a local manual upload workflow. No AI, no bank login, no cloud storage.

Current MVP screens:

1. **First Run Setup** — blank local baseline form; no sample numbers on first boot.
2. **Home Dashboard** — empty personal state until a local report exists.
3. **Upload Transactions** — local CSV/Excel/PDF bank or brokerage upload, multi-PDF merge, merchant-rule bulk fill, editable category review, and gated report generation.
4. **Monthly Report Reader** — empty personal state until a local report exists.
5. **Category Review** — empty personal state until a local upload review exists.
6. **Example Reports** — explicit fictional test-persona picker for sample dashboard/report numbers.
7. **Settings / Privacy** — built early on purpose: the local-only/safety model must be architectural, not bolted on later.

MVP shared elements: the privacy banner, verified markers, artifact trust gates,
local upload parsing, and fail-closed category/report checks. Month switching and export are
deferred; the current app keeps personal mode blank until setup/upload/report generation and writes uploaded personal artifacts only under Git-ignored local folders.

### Build order (revised from the original guess)

Original guess was Dashboard → Report Reader → Settings. Two corrections:

- **Settings/Privacy moves to #2** (trust/safety is architectural).
- **Category Review comes before the AI Memo** — it proves "engine calculates,
  suggestions only suggest, you approve", which the memo depends on.

Full order:

```text
1. First Run Setup         (blank local baseline; no prefilled numbers)
2. Home Dashboard          (personal state, no sample fallback)
3. Upload Transactions     (bank/brokerage import, category edit, merchant rules, report button)
4. Monthly Report Reader
5. Category Review         (Workbench-mode table; you approve)
6. Example Reports         (explicit fictional persona picker)
7. Stress Test Explorer    (Workbench-mode persona grid)
8. Local AI Memo           (last; optional; off by default)
```

## 4. What goes where

**Into Figma** (look, from the artifact):

- Visual system: ~10 color tokens, 4 type styles, 8pt spacing, radius/elevation.
- The 7 screen layouts.
- Trust components: privacy/status banner, verified marker, local-AI source-label.
- Prototype flow map.

**Straight into code** (logic, not look):

- Report-JSON → UI binding (done: `modules/reports/report_json.py`).
- Table rendering and sample-mode/safety gating.
- Sample-mode / safety gating and chart data.

**Back to Codex / Claude Code:**

- Use the artifact HTML as the visual spec + this JSON contract as the data spec.
- The Streamlit Read & Trust MVP is scaffolded screen-by-screen in the build order
  above.
- A custom CSS theme, reusable visual components, month switching, and export are
  later polish items, not current MVP behavior.

## 5. Figma handoff (student access)

For the Calm direction, set up:

- **Pages:** `00 Cover`, `01 Tokens`, `02 Components`, `03 Screens`, `04 Prototype`.
- **Color tokens:** background, surface, border, text-primary, text-muted,
  positive, negative, warning, accent, verified.
- **Type scale:** display serif, section serif, body sans, mono-label.
- **Spacing:** 4 / 8 / 12 / 16 / 24 / 32 / 48 (8pt base).
- **Radius/elevation:** sm/md/lg radius; one or two soft shadow levels only.
- **Components:** button, card, table row, status badge, nav, chart frame,
  empty state, privacy/status banner, verified marker, local-AI source-label.
- **Prototype flow:** Home → Report Reader sections → Settings; later Category
  Review → Stress → AI Memo.
